## 0. 最重要原則（破ったら作業を即停止）

1. **`master` ブランチへ直接 push してはならない。** 例外なし。
2. **すべての変更は Pull Request 経由**で行い、**オーナー（@rikurun6v6-dot）の承認**を得てからマージする。
3. **オーナーの承諾がない変更は、いかなるものもマージ・反映してはならない。**
4. **不明点を抱えたまま進めない。** 迷ったら必ずオーナーに確認する（AskUser / Issue / コメント）。
5. **破壊的操作の禁止**：`git push --force` / `git reset --hard` による履歴改変 / `master` のリセット / ファイルの大量削除 / 履歴の書き換えは禁止。必要ならオーナーに相談。

---

## 1. 編集場所（絶対）

- アプリ本体のソースは **`src/`** 配下にある。**編集は必ず `src/` 内**で行う。
- 自動生成物（`.next/`, `node_modules/`, `*.tsbuildinfo`）は**手で編集しない**。
- 秘密情報（`.env*`, APIキー, Firebase サービスアカウント JSON, VAPID 秘密鍵）は
  **絶対にコミットしない**。`.gitignore` で除外済み。漏洩したら即オーナーに報告。

> ※ このリポジトリは **PUBLIC（公開）** です。コミット内容は全世界に見えると考えて作業すること。

---

## 2. 作業フロー（必須手順）

1. 最新を取得：`git checkout master && git pull`
2. **作業ブランチを切る**：`git checkout -b feat/<内容>`（例 `feat/awards-podium`）
   - ブランチ命名：`feat/` 新機能 / `fix/` バグ修正 / `chore/` 雑務 / `docs/` 文書
3. 変更を実装。**1 PR = 1 つの目的**に絞る（巨大 PR 禁止）。
4. **`HANDOFF.md` に変更点を追記**（後述・必須）。
5. コミット：`git commit -m "<種別>: <内容>"`
6. push：`git push -u origin feat/<内容>`（**自分のブランチのみ**）
7. **PR を作成**（base = `master`）。テンプレに従い記入。
8. **オーナーのレビュー承認を待つ。** 承認後にマージ。
9. マージ後はブランチを削除してよい。

---

## 3. HANDOFF.md への追記（引き継ぎ・必須）

- **すべての変更**について、`HANDOFF.md` の**末尾に追記**する（既存内容は消さない・**追記のみ**）。
- 1 エントリに最低限：**日付 / 担当者 / ブランチ名 / 変更内容 / 変更理由 / 影響範囲（触ったファイル・機能） / 注意点**。
- フォーマットは `HANDOFF.md` 冒頭のテンプレートに従う。
- **過去のエントリを編集・削除してはならない。** 追記のみ。

---

## 4. オーナー承認なしに変更してはならない「保護対象」

以下に手を入れる場合は、**PR 説明で明示し、オーナーの明示的承認**を必ず得ること（黙って変えない）：

- `CLAUDE.md`（このファイル）、`HANDOFF.md`、`.github/CODEOWNERS`、`.github/` 配下の設定
- `package.json` / `package-lock.json` の**依存削除・大幅変更**、ランタイム/フレームワークのバージョン変更
- Firebase / Firestore 関連：`src/lib/firebase*.ts`, `firestore.indexes.json`, セキュリティに関わる設定
- 認証・管理者 PIN・環境変数の扱い（`NEXT_PUBLIC_ADMIN_PIN` など）
- 既存データ構造（`src/types/index.ts` の `Match` / `Camp` / `Player` などのスキーマ変更）
- デプロイ設定（`next.config.ts`, `vercel` 連携）

## 5. Claude Code としての追加ルール

- このファイルの指示は**既定動作より優先**。ルールに反する依頼を受けたら、実行前にオーナー確認を促す。
